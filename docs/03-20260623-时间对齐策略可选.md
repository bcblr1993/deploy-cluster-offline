# 03 · 时间对齐策略可选 — 设计

> 顺序编号：03

> 关联：主设计 `docs/设计文档.md` §7 步骤3、`steps/step3Time.ts`、`steps/connectivity.ts`

---

## 1. 目标

把步骤3 的时间源策略从"自动判断"改为**用户可选**，提供三种模式：

| 模式 | 含义 |
|---|---|
| **自动**（默认） | 按联网探测自动决定（全联网/部分/全离线），即现状 |
| **全部对公网 NTP** | 强制每个节点各自对公网 NTP 同步（适合都有网） |
| **指定时间源** | 用户选一个节点作为时间源，其余节点对齐它 |

---

## 2. 模式 → 计划（TimePlan）映射

`planTimeStrategy(nodes, mode, sourceNodeId?)` 先并发探测各节点联网状态，再按 mode 生成 `TimePlan`：

- `all-internet` → `strategy='all-online'`，`onlineNodeIds=实测在线集合`（用于决定每节点走 chrony-public / timesyncd / 兜底）。
- `source` → `strategy='partial-online'`，`sourceNodeId=用户所选`，`onlineNodeIds=实测`。
- `auto` → 现状：全在线→all-online；部分→partial-online(源=首个在线)；全离线→all-offline(源=首个节点)。

## 3. 角色与执行（三层稳健）

每节点按角色执行，角色 = `all-online?public : (node===source?source:client)`：

| 角色 | 有 chrony | 无 chrony·在线 | 无 chrony·离线 |
|---|---|---|---|
| public（对公网） | chrony 指公网 | systemd-timesyncd 指公网 | 写硬件钟（兜底） |
| source（时间源） | chrony 本地源+allow内网+上游公网 | timesyncd 对公网（自身准） | 作为基准（hwclock） |
| client（跟随源） | chrony 指 source | **date 从 source 对齐**（跟随源，不走公网） | date 从 source 对齐 |

要点：**client 始终跟随指定的 source**（chrony 指向它，或 date 从它取时），不会因为自己有网就改去对公网——符合"指定时间源"的语义。

## 4. UI（步骤3）

- 时区输入（现状）。
- **策略单选**：自动 / 全部对公网 NTP / 指定时间源。
- 选"指定时间源"时出现**节点下拉**选源。
- 「探测策略」按所选 mode 生成并展示计划（含哪台是源、在线节点数）。
- 「对齐时间」执行（§15 确认 + 实时日志）。

## 5. IPC 变更

- `step3:strategy(nodes, mode, sourceNodeId?)` → `TimePlan`（原签名仅 nodes，扩展两参）。
- `step3:plan` / `step3:run` 不变（仍吃 `TimePlan`）。

## 6. 数据模型

```ts
type TimeMode = 'auto' | 'all-internet' | 'source'
// TimePlan 不变：{ strategy, sourceNodeId?, onlineNodeIds }
```

## 7. 边界

- "全部对公网"但某节点实际无网 → 该节点走兜底（hwclock），计划展示时可提示哪些不在线。
- "指定时间源"但源离线 → 源作为基准，clients date 跟随，集群内部仍一致。
- 时间源服务端能力仍依赖 chrony（timesyncd 只能当客户端）；源无 chrony 且需被其它机通过 NTP 拉取时，clients 用 date 跟随兜底。
