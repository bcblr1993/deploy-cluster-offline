## 1. 目标

步骤1 支持把节点连接配置**导出为文件**、从文件**导入**。密码字段加密导出，导入时自动解密回填，免去逐台重录。

## 2. 加密方案选择

- safeStorage（本机钥匙串）密文换机器无法解密 → 不适合"导出给别人/别的机器导入"。
- **采用口令派生 + AES-256-GCM（可移植、安全）**：导出设一个口令，`scrypt(口令, salt)` 派生 32B 密钥，AES-256-GCM 加密每个密码；导入输入同一口令即一次性解密全部节点。口令错误时 GCM 校验失败 → 明确报错。

## 3. 文件格式（推荐 JSON `.json`）

```json
{
  "format": "deploy-cluster-offline/nodes",
  "version": 1,
  "exportedAt": "2026-06-23T12:00:00.000Z",
  "enc": "scrypt-aes-256-gcm",
  "nodes": [
    {
      "ip": "10.8.8.71",
      "port": 22,
      "username": "root",
      "hostname": "node-1",
      "remark": "",
      "password": "v1:<saltB64>:<ivB64>:<tagB64>:<cipherB64>"
    }
  ]
}
```

- 非敏感字段明文（便于查看/手改）；`password` 为加密令牌（每条独立 salt/iv，AES-256-GCM）。
- `format` 作判别，导入时校验。

## 4. 交互

- 步骤1 工具栏新增「导出配置」「导入配置」。
- 导出：弹口令输入 → 选保存路径 → 写文件。
- 导入：选文件 → 弹口令输入 → 解密 → 替换当前节点列表（清空旧探测结果）。口令错/格式错给明确提示。

## 5. IPC / 数据模型

- `nodes:export(nodes, passphrase)` → 保存对话框 + 加密写文件，返回路径或 null。
- `nodes:importPick()` → 打开对话框，返回文件路径或 null。
- `nodes:importDecrypt(path, passphrase)` → 读取+校验+解密，返回 `NodeConfig[]`，失败抛错。
- store 新增 `setNodes(nodes)` 整体替换。

## 6. 边界

- 口令错误：GCM 解密失败 → "口令错误或文件损坏"。
- 旧/异格式文件：`format` 不匹配 → 拒绝。
- 导入后密码在内存明文（与现有录入一致），不自动落盘明文。
