> 关联：`docs/设计文档.md` §16.4、`services/render.ts`、`pages/Step6Services.tsx`、`pages/Step4Disk.tsx`

## 1. 目标

步骤6 不仅能指定"服务装到哪个节点"，还能指定"该服务**数据落到哪个磁盘**"。复用步骤4 扫到的各节点磁盘挂载点，让用户为每个有状态实例选数据目录（默认在 `~/sprixin-iotcloud` 下；可选某挂载点；可自定义路径）。

适用服务（有数据卷）：postgres、redis、kafka、cassandra。无状态/命名卷（netdata、wechat、iotcloud 仅 logs）不在此列。

## 2. 数据流

- 步骤4 扫描结果（`Record<nodeId, DiskInfo[]>`）存入全局 store，步骤6 读取以列出该节点可用挂载点。
- 步骤6 为每个"有状态实例"提供数据位置选择：
  - **默认**：`~/sprixin-iotcloud/services/<svc>/<包内数据目录名>`（与包一致，`placement.dataPath` 留空）。
  - **某挂载点**：选某磁盘挂载点 `<mp>` → `placement.dataPath = <mp>/sprixin-iotcloud-data/<instanceId>`。
  - **自定义**：用户填绝对路径。
- 渲染引擎 `render.ts` 已支持 `placement.dataPath`：`dataVol(p)=p.dataPath || 默认相对目录`；`chmodDirs` 含该路径，部署时 `mkdir -p && chmod 777`（绝对路径原样、相对路径基于实例目录）。

## 3. 数据模型

`ServicePlacement.dataPath?: string` 已存在，承载绝对数据目录路径。新增 store：

```ts
disks: Record<string, DiskInfo[]>           // 步骤4 扫描结果（供步骤6 用）
setDisks(map)
setPlacementDataPath(instanceId, dataPath?) // 设置/清除某实例数据路径
```

## 4. UI（步骤6）

放置矩阵下方新增「数据落盘位置」区：列出每个已放置的有状态实例（instanceId · 服务 · 节点），每行一个下拉：
- 选项 = 默认 + 该节点各挂载点（来自 store.disks[nodeId]，显示挂载点与剩余/使用率）+ 自定义。
- 选挂载点即把 `dataPath` 设为 `<mp>/sprixin-iotcloud-data/<instanceId>`。
- 「生成配置预览」会反映所选路径（compose volume 用绝对路径）。

若该节点未扫描磁盘，提示去步骤4 扫描或允许手填自定义路径。

## 5. 边界

- 挂载点选错（容量不足/只读）：仅做提示，不阻断；部署时若 mkdir/chmod 失败会在日志报错。
- 集群实例分布在不同节点：各实例独立选各自节点的磁盘。
- 选了非默认路径后，家目录结构里该服务的数据目录不再在 `~/sprixin-iotcloud` 下——这是"把数据放大盘"的显式取舍，符合预期。
